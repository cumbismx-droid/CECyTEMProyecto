Primer Parcial

Módulos de un sistema informático

  • Usuarios: Son personas que se conectan al sistema para hacer uso de los servicios que este les proporciona.
  • Productos: Registra y gestiona los productos a la venta con detalles como precio, código y categoría.
  • Proveedores: Maneja la información de los proveedores que surten los productos.
  • Inventario: Controla las existencias, entradas y salidas de productos.
  • POS (Punto de Venta): Procesa las ventas, calcula totales, genera tickets y acepta pagos.
  • Reportes: Genera informes sobre ventas, inventario y actividad del sistema.
  • Tipos de usuario: Define roles y permisos dentro del sistema, como administrador y vendedor.

Diseño rápido del sistema

  • Primera pantalla: Login (usuario y contraseña).
  • Segunda pantalla: Menú principal con opciones como productos, inventario, reportes y POS.
  • Tercera etapa: Construcción del prototipo (versión funcional del sistema).

Pantalla de Login

Una pantalla de login es una interfaz que se utiliza para controlar el acceso a un sistema, aplicación o sitio web.

Sirve para:

  • Autenticar las entradas del usuario.
  • Controlar el acceso al sistema.
  • Proteger la seguridad y privacidad.
  • Asignar permisos y roles.
  • Registrar la actividad del usuario.
  • Proporcionar una experiencia personalizada.
  • Cumplir con regulaciones de seguridad y privacidad.

Componentes del login

  • Campo para ingresar el nombre de usuario.
  • Campo para ingresar la contraseña.
  • Botón para iniciar sesión.
  • Enlace para recuperar la contraseña olvidada.
  • Enlace para crear una nueva cuenta (si corresponde).

Segundo Parcial

Etapas del proceso

La metodología de desarrollo de software es un marco de trabajo utilizado para estandarizar, optimizar y controlar el proceso de desarrollo de software, asegurando productos de alta calidad.

  • Proporciona un enfoque estructurado y sistemático.
  • Permite diseñar, desarrollar, probar y mantener software.
  • Ayuda a que el proceso de desarrollo sea eficiente y eficaz.
  • Cada metodología tiene ventajas y desventajas.
  • La elección depende del proyecto, del equipo de desarrollo y de las necesidades del cliente.

Metodología de prototipos

La metodología de prototipos se basa en la creación de prototipos funcionales para facilitar la comunicación con los usuarios y permitir evaluar y mejorar el sistema antes de su desarrollo final.

  • Se centra en la interacción del usuario con el sistema.
  • Permite identificar errores y mejoras desde etapas tempranas.
  • Ayuda a validar ideas y requisitos del proyecto.
  • Favorece la retroalimentación constante de los usuarios.

Objetivo de la metodología de prototipos

El objetivo principal es crear un modelo tangible y funcional del sistema o producto, para que los usuarios y stakeholders puedan interactuar con él y proporcionar retroalimentación.

Etapas del proceso de prototipado

  • Definición del problema: Identificación del problema o necesidad que se desea resolver.
  • Creación del prototipo: Desarrollo de un prototipo inicial del sistema o producto.
  • Evaluación del prototipo: Presentación del prototipo a los usuarios y stakeholders para recopilar retroalimentación.
  • Refinamiento del prototipo: Realización de cambios y ajustes al prototipo con base en la retroalimentación recibida.
  • Iteración: Repetición de las etapas de evaluación y refinamiento hasta que el prototipo cumpla con los requisitos y expectativas.

Tercer Parcial

Pseudocódigo

El pseudocódigo es una forma de representar algoritmos utilizando un lenguaje intermedio entre el lenguaje natural y el lenguaje de programación, lo que facilita la comprensión de la lógica de un programa.

  • No depende de un lenguaje de programación específico.
  • Permite planear la solución antes de programar.
  • Facilita el análisis de problemas.
  • Ayuda a detectar errores lógicos.
  • Es fácil de leer y entender.

Objetivo del pseudocódigo

El objetivo del pseudocódigo es describir de forma clara y ordenada los pasos necesarios para resolver un problema, sin preocuparse por la sintaxis de un lenguaje de programación.

Estructuras básicas del pseudocódigo

  • Entrada: Permite recibir datos del usuario.
  • Proceso: Conjunto de instrucciones que transforman los datos.
  • Salida: Muestra los resultados obtenidos.

Estructuras de control

  • Estructura secuencial: Las instrucciones se ejecutan una tras otra.
  • Estructura condicional: Permite tomar decisiones usando condiciones (Si...Entonces...Sino).
  • Estructura repetitiva: Permite repetir instrucciones mientras se cumpla una condición (Mientras, Repetir, Para).

Ejemplo de pseudocódigo

El siguiente ejemplo muestra un pseudocódigo sencillo que solicita un número al usuario y determina si es positivo o negativo.
